## 작업 상세

### 1. 문헌 정리/포맷

S1 산출물에서 논문별 정보를 DESIGN.md 형식으로 변환:

```markdown
N. **[논문 제목]** (저자, 연도, 학회)
   - [핵심 내용 1~2문장]
   - [clau-doom 연결점]
   - 차이: [우리가 다르게 하는 부분]
```

### 2. 차별점 매핑 테이블

각 축의 대표 논문과 clau-doom의 차이를 한눈에 보여주는 테이블 작성:

```markdown
| 축 | 대표 논문 | 핵심 접근 | clau-doom 차이 |
|----|----------|----------|--------------|
| 진화 | MAP-Elites | archive 기반 QD | MD 파일 교차/변이 + LLM 정제 |
| RAG | NEC | episodic memory lookup | 구조화된 전략 문서 + 윌슨 스코어 |
| LLM-PI | AI Scientist | 단발성 자동 연구 | 세대 걸친 누적적 PI |
| FPS | VizDoom DRL | 신경망 정책 학습 | RAG 기반 검색 정책 |
```

### 3. 고유 기여점 보강

기존 4개 기여점에 각 축의 문헌 근거를 추가:

```markdown
1. **RL 없이 RAG + 경험 문서로 행동 개선**
   - NEC/MFEC의 episodic control과 유사하나, 학습 없이 문서 검색만으로 정책 형성
   - Decision Transformer가 sequence로 정책을 표현하듯, 우리는 문서로 정책을 표현
```
