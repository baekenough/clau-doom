## 크리티컬 패스

```
가장 긴 경로 (최악의 경우):
  S3-01 → S3-02 → S4-02 → S4-04

가장 짧은 경로:
  S1-01 → S4-01 (문헌 수집 → 문서 반영)
```

**병목 예상 지점**: S3 기술 검증 — 실제 코드 실행/벤치마크가 필요해서 시간이 가장 오래 걸릴 가능성. S4-04(CLAUDE.md)는 모든 결과가 모여야 하므로 마지막에 실행.

## 약한 의존성 (있으면 좋지만 없어도 진행 가능)

| 수혜 태스크 | 참조하면 좋은 태스크 | 이유 |
|------------|-------------------|------|
| S2-01 (베이스라인) | S1-04 (Doom RL 문헌) | RL 베이스라인 수치를 참조할 수 있음 |
| S2-03 (다양성 지표) | S1-01 (진화 문헌) | QD 문헌의 지표를 직접 차용 가능 |
| S3-02 (Rust 바인딩) | S3-01 (VizDoom PoC) | 기본 환경이 돌아야 바인딩 테스트 가능 |

이 약한 의존성들은 S4 통합 단계에서 반영하면 되므로 병렬 실행을 블록하지 않음.
